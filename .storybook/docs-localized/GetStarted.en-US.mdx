# Get Started

The fastest way to benefit from `enum-plus` is to define one business enum in the standard shape, then let that same enum drive rendering, lookup, and UI mapping.

## 1. Start with the standard shape

This is the most practical format for frontend work because it gives you value, label, and metadata in one place.

```ts
import { Enum } from 'enum-plus';

const StatusEnum = Enum({
  Draft: { value: 'draft', label: 'Draft', color: 'default' },
  Review: { value: 'review', label: 'In Review', color: 'processing' },
  Published: { value: 'published', label: 'Published', color: 'success' },
});
```

## 2. Query and render in business code

```ts
StatusEnum.Draft;
StatusEnum.label('review');
StatusEnum.key('published');
StatusEnum.findBy('color', 'success');
StatusEnum.toList();
StatusEnum.toMap();
```

## 3. Plug the same enum into UI components

For React + Ant Design projects, the recommended next step is `@enum-plus/plugin-antd`.

```tsx
import { Enum } from 'enum-plus';
import antdPlugin from '@enum-plus/plugin-antd';

Enum.install(antdPlugin);

const WeekEnum = Enum({
  Monday: { value: 'MON', label: 'Monday' },
  Tuesday: { value: 'TUE', label: 'Tuesday' },
});

<Select options={WeekEnum.toSelect()} />;
```

See the live demo at [Plugins / Ant Design Integration](?path=/story/plugins-ant-design-integration--playground).

## 4. Choose a localization path

- If you only need string translation in a framework-agnostic environment, use `localize` or an i18n plugin such as `@enum-plus/plugin-i18next`.
- If you want React components to refresh after language switching, use `@enum-plus/plugin-react` (for example via its `reactI18nextPlugin` export).
- Use `@enum-plus/plugin-react-i18next` when you want react-i18next integration without automatic UI refresh behavior.

See the live demo at [Plugins / React I18n](?path=/story/plugins-react-i18n--playground).
