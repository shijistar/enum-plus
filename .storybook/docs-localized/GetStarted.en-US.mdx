# Get Started

## 1. Define business enums with the standard shape first

This is the most recommended shape in the README because it gives you:

- Explicit enum values.
- A label for UI display.
- Additional metadata fields that can keep growing.

```ts
import { Enum } from 'enum-plus';

const StatusEnum = Enum({
  Draft: { value: 'draft', label: 'Draft', color: 'default' },
  Review: { value: 'review', label: 'In Review', color: 'processing' },
  Published: { value: 'published', label: 'Published', color: 'success' },
});
```

## 2. Query and render directly in business code

```ts
StatusEnum.Draft;
StatusEnum.label('review');
StatusEnum.key('published');
StatusEnum.findBy('color', 'success');
StatusEnum.toList();
StatusEnum.toMap();
```

## 3. In React + Ant Design, install the plugins

If your goal is to wire enums directly into frontend pages, the two Storybook pages below form the recommended combination:

- Plugins / Ant Design Integration: turns enums into Select, Menu, and Table filters data sources.
- Plugins / React I18n: keeps labels and enum names in sync after language switches.

Example:

```tsx
import { Enum } from 'enum-plus';
import antdPlugin from '@enum-plus/plugin-antd';

Enum.install(antdPlugin);

const WeekEnum = Enum({
  Monday: { value: 'MON', label: 'Monday' },
  Tuesday: { value: 'TUE', label: 'Tuesday' },
});

<Select options={WeekEnum.toSelect()} />;
```

## 4. Localization guidance

If you are in a cross-framework or string-only context, prefer the plain localize function. If you are in a React app and want UI to refresh automatically after language switching, prefer @enum-plus/plugin-react.
