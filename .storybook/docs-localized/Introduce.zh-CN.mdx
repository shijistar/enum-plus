# enum-plus

`enum-plus` 是一个面向运行时的枚举类库，适用于 TypeScript 和 JavaScript 项目。它保留了原生枚举直观的使用方式，同时补上了真实业务里最常见的能力：显示名称、元数据、本地化、UI 数据源生成，以及基于插件的框架集成。

## 它主要帮你消掉什么问题

很多团队真正痛的不是 enum 本身，而是围绕 enum 不断重复的运行时胶水代码：

- 反复维护 `value -> label` 映射表
- 反复手写 Select、Menu、Table filters 的数据结构
- 颜色、图标、权限等元数据散落在不同文件
- 国际化总是在后期再额外补一层映射
- 业务常量和 UI 数据结构逐渐漂移

`enum-plus` 的目标，就是把这些东西重新收拢成一个可复用的运行时 enum 对象。

## 为什么团队会持续用它

- 在业务代码里继续像原生 `enum` 一样直接使用枚举值。
- 为枚举一次性补充 label 和元数据，然后在整个项目里复用。
- 直接生成 Select、Menu、Table filters 等 UI 所需结构。
- 不再手写重复的值到展示文本映射表，并且更容易接入国际化。
- 在 React、Vue、Node.js、浏览器、小程序和 SSR 场景里共享同一份枚举定义。

## 什么时候适合用 enum-plus

当你的项目需要的不只是“编译期常量”时，`enum-plus` 就很合适：

- 业务状态需要显示名称
- 表单、菜单、表格筛选等数据结构希望从 enum 直接生成
- 枚举项需要颜色、图标、描述、权限等元数据
- 枚举名称和标签需要国际化
- 希望在多个层之间共享同一个可查询、可转换的运行时 enum 对象

如果你只需要纯粹的常量集合，不涉及运行时查询、UI 映射或本地化，那么原生 enum 或字符串联合类型也可能已经足够。
