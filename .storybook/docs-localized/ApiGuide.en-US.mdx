# API Guide

This page replaces the long README-style API walkthrough with a shorter navigation guide. For live runtime output, open the linked interactive stories next to each topic.

## 1. Pick an initialization shape

`enum-plus` supports multiple ways to build the same runtime enum experience:

- **Key-Value** — closest to native enums
- **Standard Shape** — recommended for most frontend work because it includes `value`, `label`, and metadata
- **Label Only** — useful when `value` should default to `key`
- **Array Shape** — ideal for API-driven or dynamic data
- **Native enum input** — reuse existing TypeScript enums

Start here: [Core / Enum Initialization](?path=/story/core-enum-initialization--playground)

## 2. Core instance API

| API | What it is for | Example |
| --- | --- | --- |
| `EnumValue = StatusEnum.Draft` | pick enum values directly | `StatusEnum.Draft` |
| `named` | access full enum items by key | `StatusEnum.named.Published.label` |
| `items` | iterate stable runtime items | `StatusEnum.items.map(...)` |
| `values` / `keys` / `labels` | read derived arrays | `StatusEnum.values` |
| `meta` | aggregate custom fields | `StatusEnum.meta.color` |
| `has(value)` | validate an incoming value | `StatusEnum.has(input)` |
| `findBy(field, value)` | query an item by metadata | `StatusEnum.findBy('slug', 'draft')` |
| `label(value)` / `key(value)` | display lookup and reverse lookup | `StatusEnum.label('draft')` |
| `raw(value?)` | inspect original definitions | `StatusEnum.raw()` |
| `toList(...)` / `toMap(...)` | generate app-friendly data structures | `StatusEnum.toSelect?.()` or `StatusEnum.toList()` |
| `name` | read the enum type name | `StatusEnum.name` |

Explore these in [Core / Query and Transform API](?path=/story/core-query-and-transform-api--explorer).

## 3. Naming conflicts are handled explicitly

If business fields collide with built-in API names such as `items`, `values`, or `meta`, enum item fields still win, while symbol aliases preserve the original capabilities.

Use the symbol exports when needed:

- `ITEMS`
- `KEYS`
- `VALUES`
- `LABELS`
- `META`
- `NAMED`

Live demo: [Core / Naming Conflicts](?path=/story/core-query-and-transform-api--naming-conflicts)

## 4. Static methods

| API | Purpose |
| --- | --- |
| `Enum.isEnum(obj)` | check whether a value is an enum-plus instance |
| `Enum.localize = fn` | install a global localization function |
| `Enum.extends({...})` | add shared methods to all enum instances |
| `Enum.install(plugin, options?)` | install plugins such as Ant Design or i18n integrations |

## 5. Global configuration

`Enum.config` controls cross-project defaults. The most important option is `Enum.config.autoLabel`, which helps you derive labels from `labelPrefix` rules and reduce repetitive enum declarations.

Live demo: [Core / Localization, Composition and Extension](?path=/story/core-localization-composition-and-extension--localization-and-autolabel)

## 6. TypeScript-only helpers

These are easiest to understand in the editor and source code rather than in runtime screenshots:

- `valueType`
- `keyType`
- `rawType`
- `AnyEnum`
- `GenericAnyEnum`
- `enum-plus/extension` module augmentation for custom methods

Source reference: [`src/enum.ts`](https://github.com/shijistar/enum-plus/blob/master/src/enum.ts)
