# enum-plus

`enum-plus` is a runtime-first enum library for TypeScript and JavaScript projects. It keeps the directness of native enums, then adds the things product code usually needs in practice: display labels, metadata, localization, UI data-source generation, and plugin-based framework integration.

This GitHub Pages site is now the primary long-form guide. The GitHub README stays intentionally lighter and points here when you want the full docs and demos.

## Why people use it

- Keep enum values readable in business logic, just like native `enum`.
- Attach labels and metadata once, then reuse them everywhere.
- Generate UI-friendly structures such as Select options, Menu items, and Table filters.
- Localize enum names and labels without duplicating lookup tables.
- Share one enum definition across React, Vue, Node.js, browser, mini-program, and SSR scenarios.

## When enum-plus is a good fit

Choose `enum-plus` when your project needs more than compile-time constants:

- business states that need user-facing labels
- form, table, menu, or filter data generated from enums
- metadata such as color, icon, or permission tags
- localization for enum names and labels
- a reusable runtime enum object shared across application layers

If all you need is a plain compile-time constant list with no runtime querying or UI mapping, native enums or string unions may still be enough.
