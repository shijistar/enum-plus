# 快速开始

要最快感受到 `enum-plus` 的价值，推荐先用标准格式定义一个业务 enum，然后让同一个 enum 同时承担回显、查询和 UI 映射。

## 1. 先用标准格式定义

这是前端项目里最实用的格式，因为它把 value、label 和 metadata 放在同一个地方。

```ts
import { Enum } from 'enum-plus';

const StatusEnum = Enum({
  Draft: { value: 'draft', label: '草稿', color: 'default' },
  Review: { value: 'review', label: '审核中', color: 'processing' },
  Published: { value: 'published', label: '已发布', color: 'success' },
});
```

## 2. 在业务代码里直接查询和回显

```ts
StatusEnum.Draft;
StatusEnum.label('review');
StatusEnum.key('published');
StatusEnum.findBy('color', 'success');
StatusEnum.toList();
StatusEnum.toMap();
```

## 3. 把同一个 enum 直接接到 UI 组件

对于 React + Ant Design 项目，下一步最推荐的是 `@enum-plus/plugin-antd`。

```tsx
import { Enum } from 'enum-plus';
import antdPlugin from '@enum-plus/plugin-antd';

Enum.install(antdPlugin);

const WeekEnum = Enum({
  Monday: { value: 'MON', label: '星期一' },
  Tuesday: { value: 'TUE', label: '星期二' },
});

<Select options={WeekEnum.toSelect()} />;
```

相关演示见 [Plugins / Ant Design Integration](?path=/story/plugins-ant-design-integration--playground)。

## 4. 决定本地化方案

- 如果你只需要跨框架的字符串翻译，可以直接使用 `localize` 或 `@enum-plus/plugin-i18next` 这类 i18n 插件。
- 如果你希望 React 组件在切换语言后自动刷新，推荐使用 `@enum-plus/plugin-react`（例如它导出的 `reactI18nextPlugin`）。
- 如果你只想接入 react-i18next，但不需要自动刷新 UI，可以使用 `@enum-plus/plugin-react-i18next`。

相关演示见 [Plugins / React I18n](?path=/story/plugins-react-i18n--playground)。
